# SpeakMaster 部署启动指南

## 一、环境要求

| 工具 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | Java微服务编译运行 |
| IntelliJ IDEA | 2023+ | 自带Maven，无需单独安装 |
| Python | 3.10+ | AI Service / Analysis Service |
| Node.js | 18+ | 前端构建 |
| Docker Desktop | 最新版 | 中间件容器化 |
| Ollama | 最新版 | 本地大模型推理（可选） |

## 二、启动中间件

所有中间件通过 Docker Compose 一键启动：

```bash
cd environment
docker compose up -d
```

启动后的中间件：

| 中间件 | 端口 | 账号密码 |
|--------|------|----------|
| MySQL 8.0 | 3307 | root / root |
| Redis 6.2 | 6379 | 无密码 |
| Nacos 2.2.3 | 8848(控制台) / 9848(gRPC) | nacos / nacos |
| Sentinel Dashboard | 8858 | sentinel / sentinel |
| Kafka | 9092 | - |
| Elasticsearch | 9200 | - |
| MinIO | 9000(API) / 9001(控制台) | minioadmin / minioadmin |
| Milvus | 19530 | - |
| SkyWalking OAP | 11800(gRPC) / 12800(REST) | - |
| SkyWalking UI | 8868 | - |

MySQL 启动时会自动执行 init.sql，创建 8 个数据库。

Nacos 控制台：http://localhost:8848/nacos （默认账号 nacos / nacos）。所有 Java 服务启动后会自动注册到 Nacos。

Sentinel 控制台：http://localhost:8858 （默认账号 sentinel / sentinel）。Java 服务首次被访问后会自动出现在 Sentinel 控制台中。

SkyWalking 控制台：http://localhost:8868 。查看服务拓扑、链路追踪、性能指标。

## 三、Java 后端（如果用 IDEA 操作，不需要装 Maven）

### 3.1 导入项目

IDEA 打开项目根目录，它会自动识别各个 pom.xml。

### 3.2 先编译 Common 模块（关键步骤）

所有服务都安装依赖到本地仓库：

1. 在 IDEA 右侧找到 **Maven** 面板
2. 展开 `backend/common` > **Lifecycle**
3. 双击 **install**（或右键 > Run Maven Build）（或更新所有依赖，循环符号）

这一步会把一些jar包安装到本地 Maven 仓库，其他服务才能引用到。

### 3.3 启动各微服务

common 编译完成后，每个服务都可以直接在 IDEA 里运行：

1. 找到各服务的 `*Application.java` 启动类
2. 右键 > **Run**

### 3.4 接入 SkyWalking 链路追踪（可选）

SkyWalking 通过 Java Agent 方式接入，不需要改代码：

1. 下载 SkyWalking Java Agent：https://skywalking.apache.org/downloads/ ，选择 **Java Agent**
2. 解压到项目根目录，比如 `environment/skywalking-agent/`
3. 在 IDEA 中为每个服务添加 VM Options（Run > Edit Configurations > VM options）：

```
-javaagent:环境绝对路径/environment/skywalking-agent/skywalking-agent.jar
-Dskywalking.agent.service_name=服务名
-Dskywalking.collector.backend_service=localhost:11800
```

各服务的 `service_name` 对应：

| 服务 | service_name |
|------|-------------|
| API Gateway | api-gateway |
| User Service | user-service |
| Practice Service | practice-service |
| Meeting Service | meeting-service |
| Community Service | community-service |
| Notification Service | notification-service |
| Admin Service | admin-service |
| AI Gateway | ai-gateway |

启动后在 SkyWalking UI（http://localhost:8868）可以看到服务拓扑和链路追踪。

不加 agent 也不影响服务正常运行，只是没有链路追踪数据。

**启动顺序建议**（先启动网关，再启动业务服务）：

| 顺序 | 启动类 | 端口 |
|------|--------|------|
| 1 | `ApiGatewayApplication` | 8080 |
| 2 | `UserServiceApplication` | 8081 |
| 3 | `PracticeServiceApplication` | 8082 |
| 4 | `MeetingServiceApplication` | 8083 |
| 5 | `CommunityServiceApplication` | 8084 |
| 6 | `NotificationServiceApplication` | 8086 |
| 7 | `AdminServiceApplication` | 8087 |
| 8 | `AiGatewayApplication` | 8088 |

**启动类位置**：

```
backend/api_gateway/src/.../gateway/ApiGatewayApplication.java
backend/user_service/src/.../user/UserServiceApplication.java
backend/practice_service/src/.../practice/PracticeServiceApplication.java
backend/meeting_service/src/.../meeting/MeetingServiceApplication.java
backend/community_service/src/.../community/CommunityServiceApplication.java
backend/notification_service/src/.../notification/NotificationServiceApplication.java
backend/admin_service/src/.../admin/AdminServiceApplication.java
backend/ai_gateway/src/.../ai/AiGatewayApplication.java
```

### 3.5 IDEA 多服务同时运行

IDEA 默认只允许一个 Spring Boot 实例运行。需要开启多实例（方法2比较简单，目前我用到的是方法2）：

- **方法1**：顶部菜单 Run > Edit Configurations > 勾选 "Allow parallel run"
- **方法2**：使用 IDEA 的 **Services** 面板（View > Tool Windows > Services），可以同时管理多个 Spring Boot 应用

### 3.6 IDEA 常见问题

**Q: 找不到 common 模块的依赖？**
确保先对 `backend/common` 执行了 Maven install。在 Maven 面板中：common > Lifecycle > install。

**Q: JDK 版本不对？**
File > Project Structure > Project SDK 设置为 JDK 17。

## 四、Python 服务（使用虚拟环境）

Python 依赖建议安装到虚拟环境中，避免污染全局环境。

### 4.1 AI Service（端口 8089）

```bash
cd backend/ai_service

# 创建虚拟环境（建议用conda创建）
# 激活虚拟环境 (conda)
conda activate ai_service

# 安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
或者
conda install -r requirements.txt
# 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8089
```

### 4.2 Analysis Service（端口 8085）

```bash
cd backend/analysis_service

# 创建虚拟环境
# 激活虚拟环境
# 安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
或者
conda······
# 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8085
```

### 4.3 MFA 发音分析环境

MFA 依赖较重且和其他包冲突，需要用 conda 单独隔离：
```bash
conda create -n mfa_env python=3.10 -y
conda activate mfa_env
conda install -c conda-forge montreal-forced-aligner -y
```

然后编辑 `backend/analysis_service/.env`，填入 MFA 路径：

```
MFA_PYTHON_PATH=C:/Users/你的用户名/miniconda3/envs/mfa_env/python.exe
MFA_DICTIONARY_PATH=mfa/english_us_arpa.dict
MFA_ACOUSTIC_MODEL_PATH=mfa/english_us_arpa
```

## 七、完整启动顺序总结

```
1. docker compose up -d           （中间件，含Nacos）
2. 等待Nacos启动完成               （访问 http://localhost:8848/nacos 确认）
3. IDEA 编译 common 模块并安装依赖  （Maven install）
4. IDEA 启动 8 个 Java 服务        （右键 Run，服务自动注册到Nacos）
5. 激活 venv，启动 AI Service      （端口 8089）
6. 激活 venv，启动 Analysis Service（端口 8085）
7. npm run dev 启动管理端前端      （端口 5173）
8. npm run dev 启动用户端前端      （端口 3001）
9. 启动ollama
10.环境变量配置、修改
```

## 十二、常见问题

**Q: Docker 容器启动失败？**
检查端口是否被占用：`netstat -ano | findstr :3307`

**Q: Java 服务启动报数据库连接失败？**
确认 MySQL 容器已启动且 3307 端口可访问。首次启动 JPA 会自动建表。

**Q: Python 虚拟环境激活失败（PowerShell）？**
先执行：`Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

**Q: AI 对话没响应？**
确认 Ollama 已启动 + 模型已下载 + AI Gateway(8088) 和 AI Service(8089) 都在运行。

**Q: 前端 API 请求 404？**
所有请求走 API Gateway(8080)，检查 vite.config.js 的代理配置是否指向 8080。
