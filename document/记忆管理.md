# 三层记忆管理总结

## 实现的功能

### 1. 短期记忆 (Short-term Memory)
- ✅ 从数据库查询最近N条消息
- ✅ 支持可配置的消息数量限制
- ✅ 自动按时间排序
- ✅ 区分用户消息和AI消息

### 2. 中期记忆 (Mid-term Memory)
- ✅ Elasticsearch + MySQL双写存储
- ✅ 对话总结自动生成(调用AI)
- ✅ 关键词提取和索引
- ✅ 语义搜索支持
- ✅ ES不可用时降级到MySQL
- ✅ 定时任务自动生成每日总结

### 3. 长期记忆 (Long-term Memory)
- ✅ 用户画像存储(MySQL)
- ✅ 英语水平、学习目标、兴趣话题
- ✅ 常见错误和发音弱点追踪
- ✅ 练习时长和对话次数统计
- ✅ 个性化提示词
- ✅ 自动创建默认画像

### 4. 记忆整合
- ✅ 三层记忆自动整合到AI提示词
- ✅ 构建个性化系统提示词
- ✅ 支持Meeting和Practice两种模式
- ✅ 优雅降级机制

### 5. API接口
- ✅ 获取用户画像 `GET /user/memory/profile`
- ✅ 更新用户画像 `PUT /user/memory/profile`
- ✅ 获取历史总结 `GET /user/memory/summaries`
- ✅ 生成会话总结 `POST /user/memory/summaries/generate`
- ✅ 获取最近对话 `GET /user/memory/recent`

## 核心实现逻辑

### 1. AI回复时的记忆整合流程

```java
// MeetingMessageServiceImpl.callAiGateway()
String callAiGateway(MeetingParticipant aiParticipant, List<MeetingMessage> recentMessages) {
    // 1. 提取用户ID和消息
    Long userId = extractUserId(recentMessages);
    String userMessage = extractLastUserMessage(recentMessages);
    
    // 2. 使用三层记忆构建提示词
    List<Map<String, String>> messages = memoryService.buildPromptWithMemory(
        userId,                          // 用户ID
        "meeting-" + meetingId,          // 会话ID
        "meeting",                       // 会话类型
        userMessage,                     // 当前消息
        aiParticipant.getAiRoleSetting() // AI角色设定
    );
    
    // 3. 调用AI Gateway
    Map<String, Object> response = aiGatewayClient.chat(request);
    return (String) response.get("content");
}
```

### 2. 三层记忆整合

```java
// MemoryServiceImpl.buildPromptWithMemory()
List<Map<String, String>> buildPromptWithMemory(...) {
    // 1. 获取长期记忆(用户画像)
    Map<String, Object> longTerm = getLongTermMemory(userId);
    
    // 2. 获取中期记忆(历史总结)
    List<Map<String, Object>> midTerm = getMidTermMemory(userId, currentMessage, 3);
    
    // 3. 获取短期记忆(最近消息)
    List<Map<String, Object>> shortTerm = getShortTermMemory(sessionId, sessionType, 10);
    
    // 4. 构建系统提示词
    String systemPrompt = buildSystemPrompt(longTerm, midTerm, rolePrompt);
    
    // 5. 组装消息列表
    List<Map<String, String>> messages = new ArrayList<>();
    messages.add(Map.of("role", "system", "content", systemPrompt));
    messages.addAll(shortTerm);
    messages.add(Map.of("role", "user", "content", currentMessage));
    
    return messages;
}
```

## 数据库表结构

### conversation_summary (中期记忆)
```sql
CREATE TABLE conversation_summary (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    session_id VARCHAR(100) NOT NULL,
    session_type VARCHAR(20) NOT NULL,  -- meeting/practice
    summary TEXT NOT NULL,
    keywords VARCHAR(500),
    start_time DATETIME,
    end_time DATETIME,
    message_count INT DEFAULT 0,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT DEFAULT 0,
    INDEX idx_user (user_id),
    INDEX idx_session (session_id),
    INDEX idx_type (session_type)
);
```

### user_profile (长期记忆)
```sql
CREATE TABLE user_profile (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL UNIQUE,
    english_level VARCHAR(20) DEFAULT 'intermediate',
    learning_goals TEXT,
    interests TEXT,                      -- JSON数组
    common_mistakes TEXT,                -- JSON数组
    pronunciation_weaknesses TEXT,       -- JSON数组
    learning_style VARCHAR(50),
    total_practice_minutes INT DEFAULT 0,
    total_conversations INT DEFAULT 0,
    personalized_prompt TEXT,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT DEFAULT 0
);
```

## 性能考虑

### 1. 缓存策略
- 用户画像缓存30分钟(Redis)
- 短期记忆不缓存(实时性要求高)
- 中期记忆ES查询结果缓存5分钟

### 2. 异步处理
- AI回复生成异步执行
- 对话总结生成异步执行
- 不阻塞用户消息发送

### 3. 降级方案
- ES不可用时降级到MySQL
- AI总结生成失败时使用简单摘要
- 用户画像不存在时创建默认画像
